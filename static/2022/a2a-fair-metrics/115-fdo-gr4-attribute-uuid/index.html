115-fdo-gr4-attribute-uuid

This landing page has these FAIR Signposting HTTP headers:

  • rel=cite-as to w3id PID
  • rel=type to LearningResource
  • rel=describedby to metadata.txt (text/plain)
  • rel=https://schema.org/identifier to urn:uuid:f3c157c9-0194-5a24-89a6-8c289d8a67ef (UUIDv5 hashed from the PID) with title=f3c157c9-0194-5a24-89a6-8c289d8a67ef

This is a test for FDO principle FDO-GR4: The PID record needs to contain mandatory FDO attributes, may contain optional FDO attributes and attributes agreed upon by recognized communities. Values of attributes can be references. Expectations of which attribute values are contained within the PID record and which are references pointing from the record to external sources should be specified in the PID profile or definition of said attribute in a Data Type Registry.

This test is an edge case where the link relation is an extension URI and the target is a non-resolvable (but otherwise valid) URI. It is also an example of pushing the Signposting capabilities to handle FDO key/value pairs to strings shown in title

Note: Signposting do not recommend the use of extension relations, linking to non-resolvable URIs or using link relations to express textual attributes in this way. This should rather be serialized in the metadata, e.g. using JSON-LD format.
