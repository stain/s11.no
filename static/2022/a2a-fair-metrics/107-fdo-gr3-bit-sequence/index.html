107-fdo-gr3-bit-sequence

This landing page has these FAIR Signposting HTTP headers:

  • rel=cite-as to w3id PID
  • rel=type to Dataset
  • rel=type to AboutPage
  • rel=describedby to metadata.txt (text/plain)
  • rel=item to test-apple-data.csv (text/csv)

This is a test for FDO principle FDO-GR3: If an FDO contains a structured bit-sequence, the structured PID record includes at least a reference to the location(s) where the bit-sequence encoding the content of a FAIR-DO (FDO) can be accessed and the type definition of the FDO. The structured record may also contain PIDs pointing to Metadata FDOs describing properties of the target FDO.

In this example, the rel=item points to the downloadable bit-sequence. As rel=describedby is required by FAIR Signposting level 1, but in a relaxation from FDO-GR3 it is referenced as URL as expected from FAIR Signposing, not as another PID.
